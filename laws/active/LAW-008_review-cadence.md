# LAW-008: Review Cadence

## Review Intervals (Minimum)

- **Security review**: Every 2 weeks (or at each sprint end)
- **Architecture review**: Monthly
- **Usability review**: Each sprint review during Phase 1 and beyond
- **Maintainability / tech debt review**: Each sprint planning session

## Findings Rule

Each review creates backlog items. Findings become stories with:

- Severity classification
- Due sprint target
- Linked review reference

Reviews that produce no findings must still be recorded with a "no findings" entry to demonstrate the review occurred.
