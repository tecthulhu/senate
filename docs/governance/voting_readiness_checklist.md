# Voting Readiness Checklist

Use this checklist at sprint start and before casting any vote.

## Checklist

- [ ] Membership entry is approved and listed in `MEMBERSHIP.md`.
- [ ] Canonical project identity confirmed (`org/repo`) and matches `MEMBERSHIP.md`.
- [ ] `laws/active/` present and synced from senate.
- [ ] `.senate-sync.json` exists and reflects the latest sync timestamp.
- [ ] Delegate/contact for the constituent is known.
- [ ] Vote format from LAW-012 is understood and available.
- [ ] Notification path is set up for `voting-open` issues.

## Result

- **Ready**: All checklist items are checked.
- **Not Ready**: One or more items unchecked.

## Remediation

If **Not Ready**, complete missing steps before voting:
- Update membership entry.
- Re-run sprint sync and update `.senate-sync.json`.
- Confirm canonical identity and delegate.
- Configure notifications and review LAW-012 vote format.
